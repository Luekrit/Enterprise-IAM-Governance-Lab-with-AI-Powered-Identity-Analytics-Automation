### üîß Step-by-Step: Creating `alice.hr` (Joiner Simulation)

1. **Click ‚ÄúCreate user‚Äù**  
   - Top-right orange button in AWS IAM Console.

2. **Set User Details**

   | Field                | Value                              |
   |----------------------|------------------------------------|
   | User name            | `alice.hr`                         |
   | Access type          | ‚úÖ Programmatic + Console access    |
   | Console password     | Autogenerated                      |
   | Require reset        | ‚úÖ Enabled                          |

   ‚Üí Click **Next: Permissions**

3. **Assign to Group**

   - Click **Create group**  
   - Group Name: `HR-ReadOnly`  
   - Attach Policy: `AmazonS3ReadOnlyAccess` or `S3-ReadOnlyAccess`
   - Click **Create group**  
   - Select the group and proceed

4. **Add Tags**

   | Key         | Value |
   |-------------|--------|
   | Department  | HR     |

   ‚Üí Click **Next: Review**

5. **Review & Create**

   - Username: `alice.hr`  
   - Group: `HR-ReadOnly`  
   - Policy: `AmazonS3ReadOnlyAccess` or `S3-ReadOnlyAccess`
   - Tag: `Department=HR`  
   ‚Üí Click **Create User**

6. **Save Credentials**

   - Click **Download .CSV**
   - Save console URL, username, password, and access keys

7. **Test Access**

   - Log in via AWS Console with `alice.hr`
   - Try browsing S3 buckets  
   - ‚úÖ Confirm read-only access (no upload/delete)

## üîÑ Phase 1B: Simulate Mover & Leaver in AWS IAM

---

### üîÅ MOVER: Bob changes roles (Dev ‚Üí Security Analyst)

üë®‚Äçüíª **Scenario**:  
Bob was a Developer, but is now transferring to the Security team. We need to:
- Remove him from `Developers-AWS` group  
- Add him to `Security-AWS` group  
- *(Optional)* Rotate his credentials or trigger reauthorization  

---

#### ‚úÖ Step-by-Step: Bob's Role Change

1. Go to **IAM > Users > bob.dev**
2. Click **Groups** ‚Üí Remove from `Developers-AWS`  
   - Click the ‚úñÔ∏è next to `Developers-AWS` to remove him  
3. Click **Add user to groups**  
   - Select or create `Security-AWS`  
   - Click **Add to groups**  
4. *(Optional for realism)*:  
   - Rotate Bob‚Äôs access keys (delete old + create new)  
   - Require Bob to reset his password again  

‚úÖ **Bob now has access equivalent to a Security Analyst instead of Developer.**

### üßπ LEAVER: Carol leaves the organization

üë©‚Äçüíº **Scenario**:  
Carol (Security Analyst) is leaving the company. We need to:
- Block her sign-in  
- Revoke programmatic and console access  
- Optionally delete her user record  

---

#### ‚úÖ Step-by-Step: Carol Offboarding

1. Go to **IAM > Users > carol.sec**
2. In **Security credentials** tab:  
   - Under **Console access**, click ‚Äú**Make inactive**‚Äù  
   - Under **Access keys**, click ‚Äú**Deactivate**‚Äù (or **Delete** if preferred)  

![carol-console-removal](screenshot/remove-carol-console-access4.png)

![carol-access-removal](screenshot/deactivate-carol-access-key.png)

3. Remove Carol from all groups:  
   - Click **Groups**  
   - Remove from `Security-AWS`  
![carol-group-removal](screenshot/remove-carol-from-security.png)

4. *(Optional for full cleanup)*:  
   - Click **Delete user** (if fully terminated)  

üí° *In real orgs, we'd keep his/her record for 30‚Äì90 days in a disabled state for auditing.*  
‚úÖ **Carol now has no access, and her identity is decommissioned.**
